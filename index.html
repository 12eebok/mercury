Mercury Editor


Mercury is a full featured HTML5 editor. It was built from the ground up to help your team get the most out of content editing in modern browsers.



gem 'mercury-rails'
rails generate mercury:install

Built to handle your content needs.

Built from the ground up to help your team get the most out of content editing in modern browsers.

Simple

Mercury comes bundled as a Rails Engine, so just include it in your Gemfile. Or download the current bundled package if you're not using Rails.

Unobtrusive

We don't inject javascript or css into your production pages so you're free to use whatever frameworks you want without having to worry about conflicts.

Configurable

Easily add or remove toolbar items or create entirely new tools. Any toolbar item can be tied to an action using behaviors and the command pattern.

Extendable

Full HTML, Simple, Markdown, Snippet and Image regions are supported by default, but you can just extend the base regions to build your own types.

HTML5

Built on top of the HTML5 contentEditable features, it natively supports the all the fancy new HTML5 elements, syntax, and JavaScript APIs.

Open-source

Built for and maintained by the community via GitHub. You can track or report issues and contribute code or translations, and keep up with the project.

Useful right out of the box.

Mercury comes with some predefined regions that will cover all your basic needs.

Full HTML Regions

Full regions are for full HTML markup and utilize the HTML5 contentEditable feature. These are the core of what Mercury Editor does, and provide the most flexibility and visual representation of what the content will look like when saved.

Simple Regions

Simple regions are designed for plain text. Newlines and markup are not allowed. Simple Regions are appropriate for titles, headlines, or any area where you want the conent to be editable but not the style.

Markdown Regions

Markdown regions are based on Markdown syntax (specifically the github flavored version), and aren't as full featured as the full region type. This region type is useful if you want to keep the markup clean and simple.

Snippets Regions

Snippet regions only allow snippets and don't have a concept of content editing within them. Snippets can be the way to go with complex markup and functionality, and are chunks of reusable markup that can be defined by a developer and placed into content anywhere.

Image Regions

The image region type should only be applied to an image tag. It results in the image being drag-and-drop replacable. This works well for logos and images you want to display with careful layout, and do not need any content functionality.

Custom Regions

When the provided regions don't give you what you need, create your own based on your specific content needs. This allows you to provide complex functionality that your content authors and editors might need. You'll be the most loved developer.

Browser Support

  • Firefox 4+
  • Chrome 10+
  • Safari 5+
